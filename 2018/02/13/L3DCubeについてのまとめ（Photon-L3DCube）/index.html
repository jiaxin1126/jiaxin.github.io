Photon(続き)

またphotonかーーーーーー

しょうがない。こいつをよく知らないとCubeを制御するの無理。
ただしデバイス系に本当に本当に本当に触ったことがあんまない私にとってはツラかった…いや、ツラい。
去年アメリカに行った時にちゃんと課題のRaspberry Piをいじったら良かったのに…
日本に戻ったらサーバではなくてセンサーの接続設備として使ったら良かったのに。悔しい。

まあ以上は全部ゴミ話。

Photonって何？

→公式紹介

え？英語で長くて読みたくない？
じゃあ私が説明しておくね。「説明が下手くそ」など言わないでね。

簡単に言えば、PhotonとArduinoやRaspberry Piは同じもの。
もちろん完全に同じものではない。構造的に、機能的に、値段的に違いがある。

知ってる限りではRaspberry Piはパソコンに近いやつ。
実際にもミニパソコンだね。

Arduino使ったことがないので何も言えない。

PhotonはArduinoに似てるやつ。

ではPhoton用のプログラミング言語？

Arduinoに似てるからもちろんC++で書くでしょうー
実際にも確認した。Arduinoで実行できるプログラムはPhotonにも同様に適用する。
なのでPhoton用のプログラムを書く際にCやC++で.inoファイルを書いてコンパイルし、.binファイルを生成するのは一番safe.

Processingで書き込める？

Processingで書ければいいなーJavaで書けるしPythonモードもあるらしいし。
Python Mode for Processing
個人的にJava経験が三年生時期に実験2個、C++経験は三年の夏インターンに1ヶ月しかないけれど。
情報系の学生としては恥ずかしいな…全部行けるべきなのに。

まあ話がProcessingに戻る。
結局Processingで書けるかどうかを知らない。確認したことがないから知らない。
ただしParticle-cliで実行すれば.inoと.bin限定。なのでJavaが使えないというのも基本的に確認できる。

でも！いきなりでもの話ですけれど！
L3DCubeは、Arduinoとの接続が可能です。
つまり、Arduinoを通してL3DCubeを制御するの可能だということ。
ここから本当にわからんけど、ちょっと調べてみたらProcessingとArduino間の通信が可能。
Arduino and Proccessing

まあPhotonを使わずにProcessingでArduino経由って形でL3DCubeを制御できる！かも！
いいね！

でもどう考えてもC++技能を学ぶのは一番
